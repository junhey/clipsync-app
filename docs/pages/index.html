核心特性

🪶

菜单栏常驻 · 闲置零 CPU

利用 macOS NSPasteboard.changeCount fast-path：闲置时只读一个整数， 真正的内容捕获仅在剪贴板变化时发生。实测和 Clipy 持平。

🔐

数据归你 · GitHub 私有仓库同步

存到你自己的 <owner>/clipsync 私有仓库的 data orphan 分支。content-addressed 拆 blob、自动去重，commit 永远只有 1 个。

📝

剪贴板历史 + 片段模板

自动捕获剪贴板（含图片）放进时间倒序的历史；常用模板放进 Snippet 树形管理，⌘E 打开编辑器，与 Clipy 体验一致。

⚡

直达粘贴：⌘⇧⌥1-9

不需要唤出弹窗：按一组直达快捷键，最近 N 条直接粘到光标位置。 和 Maccy / Alfred 同款体验。

🚀

开机自启 + 自动粘贴

首次启动会主动询问是否启用自启。可选「选中后自动粘贴」（需 Accessibility 权限），按 Enter 直接粘到原应用。

🪟

外观三选：菜单栏 · Dock · 两者

默认仅菜单栏（无 Dock 图标），也可切到「仅 Dock」「两者」。 实时切换，不需要重启。

🛡

来源黑名单 · 密码管理器零泄露

默认屏蔽 1Password / LastPass / KeePassXC / Bitwarden 复制的内容， 不进入历史也不上传。

📦

4.9 MB · Tauri 2 + Rust

原生体积，没有 Electron 包袱。LTO + opt-level=z 编译，9 个 Rust 单测保证核心逻辑正确。

下载与安装

⚡ 推荐

① 一键安装（Terminal）

一行命令完成下载、解压、安装到 /Applications、绕过 Gatekeeper、ad-hoc 签名、自动启动。 

curl -fsSL https://junhey.github.io/clipsync-app/install.sh | bash
看脚本干了什么
  1. 检测 macOS 平台 + 架构（aarch64 / x64）
  2. 从 GitHub Releases API 拉最新版本号 + 资产 URL
  3. 下载 .app.tar.gz 到临时目录
  4. 解压 → 复制到 /Applications/ClipSync.app
  5. xattr -dr com.apple.quarantine 解除隔离
  6. codesign --sign - ad-hoc 签名
  7. open -a ClipSync 启动

脚本源码： install.sh

② 手动下载 .app

从 GitHub Releases 下载 ClipSync_*_aarch64.app.tar.gz，拖到 /Applications。

前往 Releases → 
# 第一次启动会被 Gatekeeper 拦下，运行：
xattr -dr com.apple.quarantine /Applications/ClipSync.app

③ 自己 Build

需要 Rust 1.77+ 和 Node 20+。

git clone https://github.com/junhey/clipsync-app
cd clipsync
npm install
npm run tauri:build -- --bundles app
# 产物: src-tauri/target/release/bundle/macos/ClipSync.app

④ 配置 GitHub 同步

菜单栏 → 设置 → GitHub 同步：

  1. 在 GitHub 创建一个 private repo 名叫 clipsync（默认仓库名，可改）
  2. 生成一个有 repo scope 的 fine-grained PAT
  3. 粘到设置面板的 PAT 输入框（会存进 macOS Keychain）
  4. 60 秒后 data 分支自动出现 + 你的剪贴板已上去

FAQ

它和 Clipy / Maccy / Paste 比怎么样？

Clipy 同样优秀但不带云同步；Maccy 用 iCloud KVS（限苹果生态）；Paste 收年费且数据托管在他们服务器。 ClipSync 把同步存在你自己的 GitHub 私有仓库， 数据归你、永远不会消失、跨苹果生态都能看到。

Token 安全吗？

PAT 存在 macOS Keychain（service=ClipSync, account=github_pat）。 应用只在 sync 时读取，不持久化到任何文件。建议用 fine-grained PAT 限定权限到那个 clipsync 仓库。

仓库会不会越来越大？

不会。data 分支是 orphan branch，每次 sync force-push 一个新 root commit，仓库总大小 ≈ 当前 history 字节数（约 50 KB）。 相同内容的剪贴板项共享同一个 blob 文件，永不重复。

支持 Windows / Linux 吗？

代码是跨平台的（Tauri 2 + Rust），但目前只在 macOS 充分测试。 Windows / Linux 需要替换菜单栏图标资源 + 适配辅助功能权限的对应方案。 欢迎 PR。

怎么屏蔽某个 app 的剪贴板？

设置面板 → 隐私 → 忽略来源，每行一个 bundle id 模式（支持 * 通配符）。默认已经屏蔽 1Password / LastPass / KeePassXC / Bitwarden。

如何彻底卸载？

1. 退出 ClipSync
2. 拖 /Applications/ClipSync.app 到废纸篓
3. 删除 ~/Library/Application Support/dev.junhey.clipsync
4. 删除 macOS Keychain 中的 ClipSync / github_pat 条目
5.（可选）删除 GitHub 上的 clipsync 仓库
